# CS-255
Briefly summarize the DriverPass project. Who was the client? What type of system did they want you to design?
	
	This project was about designing a system for a company called DriverPass. The client wanted a web-based system that helps students prepare for their driver’s license. This includes online practice tests, scheduling driving lessons and tracking student progress. The system also allows instructor feedback and admin management.
What did you do particularly well?
	
	I think I did a good job organizing the system and clearly showing how each type of user interacts with it. I separated roles like admin, instructor and student, and made sure their actions were easy to follow.
If you could choose one part of your work on these documents to revise, what would you pick? How would you improve it?
	
	If I could revise one part, I would improve my UML diagrams. I would make them cleaner and add more detail, especially around things like payments and feedback. I would also improve how relationships between classes are shown so the design is easier to understand. I know there are still some things I am learning and a few areas I am not fully comfortable with yet, so I would take more time to review and refine those parts.
How did you interpret the user’s needs and implement them into your system design? Why is it so important to consider the user’s needs when designing?
	
	I used the interview and requirements to figure out what the client needed and turned those into system features like scheduling lessons, tracking progress and offering practice tests. I made sure each feature connected back to something the client asked for. 
How do you approach designing software? What techniques or strategies would you use in the future to analyze and design a system?
	
	I start by understanding the problem and breaking it into smaller parts. Then I identify the users and what they need to do, and design the system around those actions. In the future, I would continue using techniques like UML diagrams, user stories, and process flows to plan things out before building. This helps catch issues early and makes the system more organized and effective.
